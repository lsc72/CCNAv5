Résumé

Il est possible de synchroniser l'heure sur des périphériques réseau Cisco à l'aide du protocole NTP.

Les périphériques réseau Cisco peuvent consigner des messages Syslog dans une mémoire tampon interne, la console, une ligne de terminal ou un serveur Syslog externe. Un administrateur réseau peut configurer les types de messages à collecter ainsi que l'endroit où envoyer les messages horodatés.

Le protocole SNMP se compose de trois éléments, à savoir le gestionnaire, l'agent et la base de données MIB. Le gestionnaire SNMP réside sur le système de gestion de réseau (NMS), tandis que l'agent et la base de données MIB se situent au niveau des périphériques clients. Le gestionnaire SNMP peut interroger les périphériques clients afin de recueillir des informations ou utiliser un message de déroutement (TRAP) indiquant à un client de signaler immédiatement s'il atteint un seuil spécifique. Le protocole SNMP peut également être utilisé pour modifier la configuration d'un périphérique. Le protocole SNMPv3 est la version recommandée, car elle offre plus de sécurité. Le protocole SNMP est un outil de gestion d'accès distant à la fois complet et puissant. Pratiquement chaque élément disponible dans une commande show l'est également par l'intermédiaire du protocole SNMP.

Le protocole NetFlow est une technologie Cisco IOS qui est la norme pour la collecte de données opérationnelles IP à partir de réseaux IP. Le protocole NetFlow mesure efficacement quelles ressources réseau sont utilisées et pour quelle raison. Le protocole NetFlow utilise des champs d'en-tête pour pouvoir distinguer des flux de données. Le protocole NetFlow est une technologie de type « push » dans laquelle le périphérique client initie l'envoi de données vers un serveur configuré.
