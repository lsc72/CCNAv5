Encapsulation HDLC

Le résultat de la commande show interfaces serial affiche les informations spécifiques aux interfaces série. Lorsque le protocole HDLC est configuré, encapsulation HDLC doit apparaître dans le résultat, comme illustré à la Figure 1. Serial 0/0/0 is up, line protocol is up indique que la ligne est active et fonctionne. Encapsulation HDLC indique que l'encapsulation série par défaut (HDLC) est activée.

La commande show interfaces serial renvoie un état parmi six états possibles :

  • Serial x is up, line protocol is up

  • Serial x is down, line protocol is down

  • Serial x is up, line protocol is down

  • Serial x is up, line protocol is up (looped)

  • Serial x is up, line protocol is down (disabled)

  • Serial x is administratively down, line protocol is down

Sur les six états possibles, cinq indiquent un problème. La Figure 2 présente ces cinq états, le problème associé à chaque état, et indique comment résoudre ce problème.

La commande show controllers est un autre outil de diagnostic important dans le cadre du dépannage de lignes série, comme illustré à la Figure 3. La sortie renvoyée indique l'état des canaux de l'interface et signale la présence ou l'absence d'un câble. Dans la figure, l'interface série 0/0/0 comporte un câble DCE V.35 connecté. La syntaxe de la commande est variable, en fonction de la plateforme. Les routeurs Cisco série 7000 utilisent une carte de contrôleur cBus pour connecter les liaisons série. Avec ces routeurs, utilisez la commande show controllers cbus.

Si la sortie de l'interface électrique affiche UNKNOWN au lieu de V.35, EIA/TIA-449 ou un autre type d'interface électrique, le problème vient sans doute d'un câble qui n'est pas correctement branché. Il est également possible que le câblage interne de la carte présente un problème. Si l'interface électrique est inconnue, l'affichage correspondant à la commande show interfaces serial présente l'interface et le protocole comme étant désactivés.
