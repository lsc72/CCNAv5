Configuration de la traduction d'adresses réseau (NAT)

Configuration de la fonction NAT statique

La commande show ip nat translations permet de vérifier le fonctionnement de la NAT. Elle affiche les traductions NAT actives. Les traductions statiques, contrairement aux traductions dynamiques, se trouvent toujours dans la table NAT. La Figure 1 présente le résultat de cette commande dans l'exemple de configuration précédent. Comme l'exemple est une configuration NAT statique, la traduction est toujours présente dans la table NAT, qu'il y ait ou non des communications actives. Si la commande est émise pendant une session active, le résultat indique également l'adresse du périphérique externe comme à la Figure 1.

La commande show ip nat statistics est également utile. Comme le montre la Figure 2, la commande show ip nat statistics affiche des informations sur le nombre total de traductions actives, les paramètres de configuration NAT, le nombre d'adresses dans le pool et le nombre d'adresses attribuées.

Pour vérifier que la traduction NAT fonctionne, il est préférable d'effacer les statistiques des traductions passées à l'aide de la commande clear ip nat statistics avant d'effectuer le test.

Avant toute communication avec le serveur Web, la commande show ip nat statistics n'affiche aucune occurrence. Une fois que le client a établi une session avec le serveur Web, la commande show ip nat statistics présente cinq occurrences. Cela permet de vérifier que la traduction NAT statique a lieu sur R2.
