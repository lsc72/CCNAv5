Implémentations de VLAN

Dépannage des VLAN et des trunks

S'il n'y a toujours aucune connexion entre les périphériques d'un VLAN, mais que les problèmes d'adressage IP ont été réglés, consultez le diagramme de la Figure 1 pour le dépannage :

Étape 1. Utilisez la commande show vlan pour vérifier que le port appartient au VLAN prévu. Si le port est attribué au mauvais VLAN, utilisez la commande switchport access vlan pour corriger l'appartenance. Utilisez la commande show mac address-table pour vérifier que les adresses ont été apprises sur un port spécifique du commutateur et le VLAN auquel le port est attribué.

Étape 2. Si le VLAN auquel est affecté le port est supprimé, le port devient inactif. Utilisez la commande show vlan ou show interfaces switchport.

Pour afficher la table d'adresses MAC, utilisez la commande show mac-address-table. L'exemple de la Figure 2 affiche les adresses MAC qui ont été acquises sur l'interface F0/1. Vous constatez que l'adresse MAC 000c.296a.a21c a été acquise sur l'interface F0/1 du VLAN 10. Si ce chiffre n'est pas le numéro VLAN voulu, modifiez l'appartenance des ports à l'aide de la commande switchport access vlan.

Chaque port d'un commutateur appartient à un VLAN. Si le VLAN auquel appartient le port est supprimé, le port devient inactif. Tous les ports appartenant au VLAN qui a été supprimé ne peuvent plus communiquer avec le reste du réseau. Utilisez la commande show interface f0/1 switchport pour vérifier si le port est inactif. Si le port est inactif, il n'est pas opérationnel tant que le VLAN manquant n'est pas créé à l'aide de la commande vlan vlan_id.
