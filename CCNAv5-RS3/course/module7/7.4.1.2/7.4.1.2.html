Configuration du protocole EIGRP pour IPv6

Protocole EIGRP pour IPv4 versus IPv6

Voici une comparaison des principales fonctionnalités des protocoles EIGRP pour IPv4 et EIGRP pour IPv6 :

  • Routes annoncées : le protocole EIGRP pour IPv4 annonce des réseaux IPv4, tandis que le protocole EIGRP pour IPv6 annonce des préfixes IPv6.

  • Vecteur de distance : le protocole EIGRP pour IPv4 et le protocole EIGRP pour IPv6 sont des protocoles de routage à vecteur de distance avancés. Les deux protocoles utilisent les mêmes distances administratives.

  • Technologie de convergence : le protocole EIGRP pour IPv4 et le protocole EIGRP pour IPv6 utilisent l'algorithme DUAL. Les deux protocoles utilisent les mêmes techniques et processus DUAL, notamment le successeur, le successeur potentiel, la distance de faisabilité et la distance annoncée.

  • Métrique : le protocole EIGRP pour IPv4 et le protocole EIGRP pour IPv6 utilisent tous deux la bande passante, le délai, la fiabilité et la charge pour leur métrique composite. Les deux protocoles de routage utilisent la même métrique composite et utilisent uniquement la bande passante et le délai, par défaut.

  • Protocole de transport : le protocole de transport fiable (RTP, Reliable Transport Protocol) garantit l'acheminement des paquets EIGRP à tous les voisins pour les deux protocoles, le protocole EIGRP pour IPv4 et le protocole EIGRP pour IPv6.

  • Messages de mise à jour : le protocole EIGRP pour IPv4 et le protocole EIGRP pour IPv6 envoient des mises à jour incrémentielles en cas de modification de l'état d'une destination. Les termes « partiel » et « limité » sont utilisés dans les deux protocoles pour désigner les mises à jour.

  • Mécanisme de détection de voisins : le protocole EIGRP pour IPv4 et le protocole EIGRP pour IPv6 utilisent un simple mécanisme Hello pour identifier les routeurs voisins et établir des contiguïtés.

  • Adresses source et de destination : le protocole EIGRP pour IPv4 envoie des messages à l'adresse de multidiffusion 224.0.0.10. Ces messages utilisent l'adresse IPv4 source de l'interface sortante. Le protocole EIGRP pour IPv6 envoie ses messages à l'adresse de multidiffusion FF02::A. Les messages EIGRP pour IPv6 sont fournis à l'aide de l'adresse link-local IPv6 de l'interface de sortie.

  • Authentification : le protocole EIGRP pour IPv4 peut utiliser l'authentification en clair ou l'authentification Message Digest 5 (MD5). Le protocole EIGRP pour IPv6 utilise MD5.

  • ID de routeur : le protocole EIGRP pour IPv4 et le protocole EIGRP pour IPv6 utilisent tous deux un nombre 32 bits comme ID de routeur EIGRP. L'ID de routeur 32 bits est représenté par une notation décimale à point et correspond généralement à une adresse IPv4. Si le routeur EIGRP pour IPv6 n'a pas été configuré avec une adresse IPv4, vous devez utiliser la commande eigrp router-id pour configurer un ID de routeur 32 bits. Le processus de détermination de l'ID de routeur est le même pour le protocole EIGRP pour IPv4 et le protocole EIGRP pour IPv6.
