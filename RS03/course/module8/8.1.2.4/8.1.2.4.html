Configurations EIGRP avancées

Résumé manuel

Alors que la récapitulation automatique n'est pas disponible pour les réseaux EIGRP IPv6, il est possible d'activer la récapitulation manuelle pour le protocole EIGRP IPv6.

La Figure 1 présente une topologie EIGRP IPv6 avec quatre adresses de bouclage configurées sur R3. Ces adresses virtuelles servent à représenter les réseaux physiques dans la table de routage IPv6 de R3. Ces réseaux peuvent être récapitulés manuellement dans le protocole EIGRP pour IPv6.

La Figure 2 présente la configuration des adresses de bouclage IPv6 sur R3. Seules quatre adresses de bouclage sont présentées dans la topologie et configurées sur R3 ; toutefois, dans cet exemple, on suppose que tous les sous-réseaux 2001:DB8:ACAD::/48 peuvent être atteints via R3.

Pour configurer la récapitulation manuelle du protocole EIGRP pour IPv6 sur une interface EIGRP spécifique, utilisez la commande de mode de configuration d'interface suivante :

Router(config-if)# ipv6 summary-address eigrp as-number prefix/prefix-length

La Figure 3 montre la configuration permettant de propager une route récapitulative manuelle EIGRP pour IPv6 vers R1 et R2 pour le préfixe 2001:DB8:ACAD::/48. Comme pour le protocole EIGRP pour IPv4, R3 comprend une route récapitulative vers null0 sous forme de mécanisme de prévention des boucles.

Il est possible de vérifier la réception de la route récapitulative manuelle en examinant la table de routage des autres routeurs du domaine de routage. La Figure 4 montre la route 2001:DB8:ACAD::/48 dans la table de routage IPv6 de R1.
