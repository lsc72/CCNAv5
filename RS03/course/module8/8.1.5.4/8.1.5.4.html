Configurations EIGRP avancées

EIGRP sécurisé

Une fois l'authentification des messages EIGRP configurée sur un routeur, les voisins adjacents qui n'ont pas encore été configurés pour l'authentification ne sont plus considérés comme des voisins EIGRP. Par exemple, si l'interface Serial 0/0/0 de R1 a été configurée pour l'authentification MD5, mais si R2 n'a pas encore été configuré, le message IOS suivant apparaît sur R1 :

%DUAL-5-NBRCHANGE: EIGRP-IPv4 1: Neighbor 172.16.3.2 (Serial0/0/0) is down: authentication mode changed

Lorsque l'interface Serial 0/0/0 adjacente sur R2 est configurée, la contiguïté est rétablie et le message IOS suivant s'affiche sur R1 :

%DUAL-5-NBRCHANGE: EIGRP-IPv4 1: Neighbor 172.16.3.2 (Serial0/0/0) is up: new adjacency

Les mêmes messages s'affichent également sur R2.

Les contiguïtés se forment uniquement lorsque les deux périphériques connectés sont configurés pour l'authentification, comme indiqué à la Figure 1. Afin de vérifier que les contiguïtés EIGRP correctes ont été formées après avoir été configurées pour l'authentification, exécutez la commande show ip eigrp neighbors sur chaque routeur. La Figure 2 montre que les trois routeurs ont rétabli les contiguïtés de voisinage après avoir été configurés pour l'authentification EIGRP.

Pour vérifier les contiguïtés de voisinage EIGRP pour IPv6, utilisez la commande show ipv6 eigrp neighbors.
