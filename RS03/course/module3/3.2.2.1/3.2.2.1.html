La configuration de l'agrégation de liaisons

Vérification et dépannage d'un EtherChannel

Il existe plusieurs commandes permettant de vérifier une configuration EtherChannel. Tout d'abord, la commande show interface port-channel affiche l'état général de l'interface de port-channel. Dans la Figure 1, l'interface Port Channel 1 est active.

Lorsque plusieurs interfaces de canal de port sont configurées sur le même périphérique, utilisez la commande show etherchannel summary pour afficher une ligne d'informations unique par canal de port. Dans la Figure 2, le commutateur a un EtherChannel configuré ; le groupe 1 utilise LACP.

Le groupement d'interfaces est constitué des interfaces FastEthernet0/1 et FastEthernet0/2. Le groupe est un EtherChannel de couche 2, qui est en fonctionnement, comme indiqué par les lettres SU à côté du numéro de canal de port.

Utilisez la commande show etherchannel port-channel pour afficher des informations concernant une interface port-channel spécifique, comme illustré dans la Figure 3. Dans l'exemple, l'interface Port Channel 1 est constituée de deux interfaces physiques, FastEthernet0/1 et FastEthernet0/2. Elle utilise LACP en mode actif. Elle est correctement connectée à un autre commutateur avec une configuration compatible, c'est pourquoi le canal est indiqué comme étant en fonctionnement.

Sur toute interface physique d'une liaison EtherChannel, la commande show interfaces etherchannel peut fournir des informations sur le rôle de l'interface dans l'EtherChannel, comme illustré dans la Figure 4. L'interface FastEthernet 0/1 fait partie du groupe EtherChannel 1. Le protocole pour cet EtherChannel est LACP.
