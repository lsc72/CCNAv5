Dépannage du protocole Frame Relay

Si la procédure de vérification indique que votre configuration Frame Relay ne fonctionne pas correctement, l'étape suivante est le dépannage de la configuration.

Utilisez la commande debug frame-relay lmi pour déterminer si le routeur et le commutateur Frame Relay envoient et reçoivent correctement les paquets LMI.

Examinez l'illustration pour voir le résultat d'un échange LMI.

  • out est un message d'état LMI envoyé par le routeur.

  • in est un a message reçu du commutateur Frame Relay.

  • Le message d'état LMI complet est de type 0.

  • L'échange LMI est de type 1.

  • dlci 102, status 0x2 indique que le DLCI 102 est actif.

Les valeurs possibles du champ d'état sont les suivantes :

  • 0x0 : le commutateur a programmé le DLCI, mais il est inutilisable. Il est possible que l'autre extrémité du circuit virtuel permanent ne fonctionne pas.

  • 0x2 : le commutateur Frame Relay dispose du DLCI et tout fonctionne.

  • 0x4 : le commutateur Frame Relay n'a pas ce DLCI programmé pour le routeur, mais il a été programmé dans le passé. Ceci est peut-être causé par l'inversion des DLCI sur le routeur ou parce que le circuit virtuel permanent a été supprimé par l'opérateur dans le nuage Frame Relay.
