Dépannage avec une approche systématique

Documentation du réseau

Pour que les administrateurs réseau soient en mesure de surveiller et de dépanner un réseau, ils doivent posséder une documentation à la fois complète, précise et actuelle du réseau. Cette documentation comprend :

  • les fichiers de configuration, y compris les fichiers de configuration réseau et les fichiers de configuration du système terminal 

  • les schémas de topologie physique et logique 

  • un niveau de performances de référence

La documentation réseau permet aux administrateurs de diagnostiquer et de résoudre efficacement les problèmes réseau, sur la base de la conception du réseau et des performances attendues en conditions de fonctionnement normales. La totalité de la documentation réseau doit être conservée dans un emplacement unique, soit sous forme de copie papier soit sur un serveur protégé du réseau. La documentation de sauvegarde doit quant à elle être conservée à un autre endroit.

Fichiers de configuration réseau

Les fichiers de configuration réseau contiennent des enregistrements précis et à jour du matériel et des logiciels utilisés dans un réseau. Les fichiers de configuration réseau doivent inclure une table pour chaque périphérique réseau, reprenant toutes les informations pertinentes à propos de ce périphérique. La Figure 1 illustre des exemples de tables de configuration réseau pour deux routeurs. La Figure 2 montre une table similaire pour un commutateur LAN.

Les informations pouvant être capturées dans une table de périphérique sont les suivantes :

  • le type de périphérique et la désignation du modèle 

  • le nom de l’image IOS 

  • le nom d’hôte du réseau du périphérique 

  • l’emplacement du périphérique (bâtiment, étage, salle, bâti, panneau) 

  • s’il s’agit d’un périphérique modulaire, tous les types de module et l’emplacement de module où ils se situent 

  • les adresses de couche liaison de données 

  • les adresses de couche réseau 

  • toute information importante supplémentaire sur les aspects physiques du périphérique

Fichiers de configuration du système terminal

Les fichiers de configuration du système terminal concernent principalement le matériel et les logiciels utilisés au niveau des périphériques du système terminal, tels que les serveurs, les consoles de gestion du réseau et les stations de travail utilisateur. Un système terminal incorrectement configuré peut nuire aux performances globales d'un réseau. Pour cette raison, la possession d'un exemple d'enregistrement de planification initiale du matériel et des logiciels utilisés sur les périphériques et figurant dans la documentation du système terminal, comme le montre la Figure 3, peut s'avérer très utile lors du dépannage.

Pour les besoins du dépannage, les informations suivantes doivent être documentées dans la table de configuration du système terminal :

  • le nom du périphérique (fonction) 

  • le système d’exploitation et la version 

  • les adresses IPv4 et IPv6 

  • le masque de sous-réseau et la longueur de préfixe 

  • les adresses de la passerelle par défaut, du serveur de noms de domaine (DNS) et du serveur WINS 

  • toute application réseau à haut débit exécutée par le système terminal
