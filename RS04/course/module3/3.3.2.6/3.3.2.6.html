Configuration de PPP

Authentification PPP

La procédure mentionnée dans le tableau décrit comment configurer l'encapsulation PPP et les protocoles d'authentification PAP/CHAP. Il est essentiel de procéder à une configuration correcte, car PAP et CHAP s'appuient sur ces paramètres pour leur authentification.

Configuration de l'authentification PAP

La figure 1 présente un exemple de configuration d'authentification PAP bidirectionnelle. Les deux routeurs authentifient et sont authentifiés, de telle sorte qu'un effet de miroir se produit sur les commandes d'authentification PAP. Le nom d'utilisateur et le mot de passe PAP que chaque routeur envoie doit correspondre à ceux spécifiés dans la commande username name password password de l'autre routeur.

Le protocole PAP procure une méthode simple permettant à un nœud distant d'établir son identité à l'aide d'un échange en deux étapes. Ceci s'effectue uniquement à l'établissement initial de la liaison. Le nom d'hôte du routeur doit correspondre au nom d'utilisateur que l'autre routeur a configuré pour le protocole PPP. Les mots de passe doivent également correspondre. Pour spécifier les paramètres de nom d'utilisateur et de mot de passe, utilisez la commande suivante : ppp pap sent-username name password password.

Utilisez le contrôleur de syntaxe de la Figure 2 pour configurer l'authentification PAP sur l'interface série 0/0/1 du routeur R1.

Configuration de l'authentification CHAP

CHAP vérifie régulièrement l'identité du nœud distant à l'aide d'une connexion en trois étapes. Le nom d'hôte d'un routeur doit correspondre au nom d'utilisateur que l'autre routeur a configuré. Les mots de passe doivent également correspondre. La vérification a lieu lors de l'établissement initial de la liaison et peut être répétée à tout moment une fois la liaison établie.
