Dépannage de PPP

Souvenez-vous que la commande debug sert au dépannage. Elle est accessible depuis le mode d'exécution privilégié de l'interface en ligne de commande. Le résultat de debug affiche des informations sur le fonctionnement des routeurs, le trafic généré par le routeur ou reçu par le routeur, ainsi que tous les messages d'erreur. Il peut par conséquent utiliser une grande quantité de ressources, obligeant ainsi le routeur à commuter les paquets en cours de débogage. La commande debug ne doit pas être utilisée comme outil de surveillance. Elle est faite pour être utilisée pendant un laps de temps réduit, pour le dépannage.

Utilisez la commande debug ppp pour afficher des informations sur le fonctionnement du protocole PPP. La figure ci-contre affiche la syntaxe de la commande. Utilisez la forme no de cette commande pour désactiver la sortie de débogage.

Utilisez la commande debug ppp lorsque vous recherchez les éléments suivants :

  • Les NCP pris en charge aux deux extrémités d'une connexion PPP

  • Les boucles dans un interréseau PPP

  • Les nœuds qui négocient correctement les connexions PPP (ou ceux qui ne le font pas)

  • Les erreurs sur la connexion PPP

  • Les causes d'échec de la session CHAP

  • Les causes d'échec de la session PAP

  • Les informations spécifiques à l'échange de connexions PPP avec le protocole CBCP (Callback Control Protocol), utilisé par les clients Microsoft

  • Les informations sur le numéro de séquence des paquets incorrects lorsque la compression MPPC est activée
