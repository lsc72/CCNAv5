Configuration de la traduction d'adresses réseau (NAT)

Redirection

La figure intègre la fenêtre de configuration de la redirection d'un seul port affiché sur un routeur de petite structure Linksys EA6500. Par défaut, la redirection n'est pas activée sur le routeur.

Pour activer la redirection pour les applications, il convient de spécifier l'adresse locale interne vers laquelle les demandes doivent être redirigées. Sur la figure, les demandes de service HTTP arrivant sur ce routeur Linksys sont transférées vers le serveur Web dont l'adresse locale interne est 192.168.1.254. Si l'adresse IPv4 du réseau étendu externe du routeur de petite structure est 209.165.200.225, l'utilisateur externe peut saisir http://www.example.com. Le routeur Linksys redirige alors la requête HTTP vers le serveur Web interne à l'adresse IPv4 192.168.1.254, en utilisant le numéro de port par défaut 80.

Il est possible d'indiquer un port différent du port par défaut 80. Cependant, l'utilisateur externe doit connaître le numéro de port à utiliser. Pour indiquer un autre port, il faut modifier la valeur du port externe (External Port) dans la fenêtre Single Port Forwarding.

La méthode à suivre pour configurer la redirection dépend de la marque et du modèle du routeur haut débit utilisé sur le réseau. Il existe cependant des étapes communes à effectuer. Si les instructions fournies par le FAI ou accompagnant le routeur ne contiennent pas les instructions appropriées, le site Web http://www.portforward.com propose les guides relatifs à plusieurs routeurs haut débit. Vous pouvez suivre les instructions pour ajouter ou supprimer des ports afin de répondre aux besoins des applications que vous voulez autoriser ou refuser.
