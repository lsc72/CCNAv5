Configuration du protocole OSPFv3 à zone unique

Configuration du OSPFv3

Les ID de routeur doivent parfois être modifiés, par exemple, si l'administrateur réseau a défini un nouveau modèle d'identification d'ID de routeur. Cependant, une fois qu'un routeur OSPFv3 définit un ID de routeur, cet ID ne peut pas être modifié tant que le routeur n'est pas redémarré ou que le processus OSPF n'est pas supprimé.

Dans la Figure 1, notez que l'ID de routeur actuel correspond à 10.1.1.1. L'ID de routeur OSPFv3 devrait être 1.1.1.1.

Dans la Figure 2, l'ID de routeur 1.1.1.1 est attribué à R1.

Remarque : pour réinitialiser l'ID de routeur, il est préférable de supprimer le processus OSPF.

Dans la Figure 3, le processus de routage OSPF est supprimé à l'aide de la commande du mode d'exécution privilégié clear ipv6 ospf process. Ceci contraint le protocole OSPF sur R1 à renégocier les contiguïtés de voisinage au moyen du nouvel ID de routeur.

La commande show ipv6 protocols vérifie que l'ID de routeur a changé.
